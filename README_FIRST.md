# Read This First

TSA adalah Engineering Operating System untuk mengatur cara AI bekerja pada project.

Namun TSA tidak boleh selalu dijalankan secara penuh.

Untuk menghemat token dan menjaga eksekusi tetap cepat, setiap task wajib memilih Runtime Mode terlebih dahulu.

---

# Runtime Mode

```text
Mode 0 — Direct
Mode 1 — Micro Patch
Mode 2 — Feature Patch
Mode 3 — Architecture / Foundation
Mode 4 — Review Only
```

---

# Mode Selection Rule

Gunakan mode paling ringan yang cukup untuk menyelesaikan task.

Jangan memakai Mode 3 jika task cukup diselesaikan dengan Mode 1.

---

# Quick Selector

```text
Jawaban cepat / non-project
→ Mode 0

Bug kecil / edit 1–2 file
→ Mode 1

Fitur kecil-menengah
→ Mode 2

Project baru / blueprint / perubahan besar
→ Mode 3

Audit hasil agent / review tanpa coding
→ Mode 4
```

---

# Core Reminder

TSA wajib mengikuti prinsip:

- Deliver Value
- Keep It Simple
- Read Less, Think More
- Execution First
- Token Efficiency
- Stop When Objective Completed
